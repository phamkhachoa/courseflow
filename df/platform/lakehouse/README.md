# Lakehouse Platform

Lakehouse assets are split into Bronze, Silver and Gold layers.

## Layer Rules

- Bronze: raw immutable events and CDC changes. Minimal parsing, full replay value.
- Silver: conformed, deduplicated, typed and PII-classified data.
- Gold: business-ready marts, metrics and ML feature datasets.

## Production Requirements

- Use event-time partitioning and preserve ingestion-time metadata.
- Record source offset/snapshot for every row lineage.
- Apply the Iceberg naming, partitioning and table maintenance standard in
  [iceberg-table-standards.md](iceberg-table-standards.md).
- All Gold writes must be atomic and quality-gated.
- Production-like Silver/Gold commits must emit `lakehouse_snapshot_evidence.v1`, generated by
  `enterprise-df snapshot-evidence-record`, so release gates can verify Iceberg snapshot metadata,
  schema/partition hashes, row counts, content hashes and upstream offset ledger binding.
- Backfills must be reproducible from Bronze snapshots.

4. P0 còn chặn production

P0 Chưa có service-to-service security và network isolation

Các service nghiệp vụ hiện dựa vào CurrentUser từ header gateway. Nếu service bị gọi trực tiếp trong network nội bộ với header tự dựng, nhiều service vẫn sẽ tin header. Identity-service đã tự verify JWT, nhưng các service còn lại chưa có JWT verification filter, mTLS, internal network policy, hoặc shared gateway-only ingress enforcement.

Cần làm: chỉ expose service qua gateway trong K8s NetworkPolicy; thêm service-level JWT verification hoặc mTLS; tách public user token và internal service token; deny direct traffic không qua gateway.

P0 Authorization còn thiếu entitlement theo course/org

Role/ownership đã tốt hơn, nhưng nhiều endpoint vẫn chỉ cần "authenticated user": đọc quiz/course material/live session/video manifest có thể chưa kiểm tra user đã enroll course hay thuộc org/section đó. LMS production phải kiểm tra ít nhất: enrollment active, section membership, instructor owns course/section, org/dept scope, paid access/license.

Cần làm: chuẩn hóa policy engine gọi identity/authz + enrollment/course scope; mọi read nội dung khóa học phải check canAccessCourse(user, courseId).

P0 Test coverage chưa đủ để tin production

Mới có 4 test class: gateway JWT, enrollment service, gradebook aggregation, peer-review aggregate. Đây là tiến bộ so với bản cũ, nhưng còn rất xa chuẩn production. Chưa có controller security tests cho các endpoint vừa harden, chưa có Testcontainers, chưa có Kafka/outbox integration, chưa có migration smoke test.

Cần làm: bắt buộc thêm test cho authz matrix, quiz timing/reveal answers, enrollment concurrency, gradebook event-to-final-grade, media permission, outbox retry/idempotency.

P0 Chưa có pipeline triển khai production

Repo hiện có backend/infra/docker/docker-compose.yml cho local infra, nhưng chưa có Dockerfile cho từng service, chưa có K8s/Helm, chưa có CI/CD, chưa có image scanning, migration pipeline, rollback plan, secret manager.

Cần làm: multi-stage Dockerfile non-root cho service; GitHub Actions/GitLab CI build-test-scan; Helm chart; probes, limits, HPA/PDB; Liquibase migrate job; separate dev/staging/prod config.

P0 Event backbone chưa có DLQ/observability

Outbox relay đã rộng hơn, nhưng production cần biết event nào kẹt, consumer lag bao nhiêu, retry đến đâu, message độc bị đưa sang đâu. Hiện chưa thấy DLQ/parking-lot, retry budget theo loại lỗi, schema registry/versioning bắt buộc, metric/alert cho outbox lag.

Cần làm: Kafka DLQ per consumer, relay parking-lot, outbox partial index + purge, idempotency key chuẩn, event-contracts dùng thật trong producer/consumer, dashboard lag.

5. Review các service chính

Identity Service + API Gateway

Đã ổnCần làm tiếp
Gateway tồn tại, route đầy đủ, strip X-User-*, forward verified identity. Identity tự verify JWT, fail-fast secret, refresh reuse revoke-all, login lockout cơ bản. Chuyển HS256 sang RS256/ES256 + JWKS/kid rotation; password reset, email verification, MFA/WebAuthn; device/session management; rate limit login; audit log security; security headers; token revocation strategy cho access token.

Course / Authoring / Enrollment / Organization

Đã ổnCần làm tiếp
Course CRUD và authoring workflow đã có owner/admin guard; enrollment có capacity, state machine, waitlist promotion, audit/outbox; organization read đã yêu cầu authenticated, create section admin/operator. Course access policy theo enrollment/section; prerequisite enforcement; content authoring API đủ giàu (module item type, publishing snapshot immutable, rollback version); course completion event; enrollment concurrency test bằng Postgres lock thật.

Assignment / Quiz / Gradebook / Peer Review

Đã ổnCần làm tiếp
Student không tự chấm điểm; submission/gradebook có self-or-staff; quiz student view sanitized; manual grade staff-only; gradebook weight/category/grading scheme; peer-review final score server-side; MapStruct đã thay mapping chính. Quiz attempt snapshot thật khi start, autosave endpoint, auto-submit khi timeout/grace period, lock attempt concurrent submit; assignment late policy/rubric audit; grade override audit và regrade history; question bank/tag/random draw; plagiarism/LTI/proctoring integration nếu cạnh tranh hệ lớn.

Media / Notification / Analytics / Review / Discussion / Live Session

Đã ổnCần làm tiếp
Media asset owner check, upload content-type whitelist, HMAC playback token; notification self-only inbox/preferences; analytics self-or-staff; discussion/review authorId lấy từ caller; live-session host/register lấy từ caller. Video/content entitlement theo enrollment/course; notification fan-out theo enrollment/role và retry; analytics privacy/aggregation threshold; review abuse/rate limit/moderation queue; live-session provider webhook verification và attendance reconciliation.

6. Kiến trúc cần hoàn thiện để cạnh tranh LMS lớn

So với Moodle / Canvas / Open edX

  • Content authoring: cần rich content blocks, quiz question bank, reusable learning objects, import/export chuẩn IMS Common Cartridge/SCORM/xAPI/LTI.
  • Learning delivery: cần enrollment/section/cohort/group policy xuyên service, prerequisites, drip schedule, completion rules, transcript.
  • Assessment: cần rubric versioning, grade override audit, moderation workflow, accommodations, question randomization theo bank/tag, attempt snapshot immutable.
  • Integrations: LTI 1.3, SSO/OIDC/SAML, SIS sync, calendar/email integration, webhooks/public API, SCIM nếu đi enterprise.
  • Operations: multi-tenant isolation, backup/restore, audit trail đầy đủ, data retention, FERPA/GDPR controls, accessibility WCAG 2.2 AA.

8. Kiểm chứng đã chạy

  • Scan persistence: không thấy JdbcTemplate, NamedParameterJdbcTemplate, JdbcClient, nativeQuery=true, createNativeQuery trong domain services. Ngoại lệ còn lại là outbox-relay, hợp lý vì đây là poller hạ tầng đọc nhiều DB.
  • Scan mapper: 18 @Mapper(componentModel = "spring") trong service mapper.
  • Module test đã chạy xanh sau các hardening mới: mvn test -q -pl services/course-service,services/enrollment-service,services/announcement-service,services/analytics-service,services/deadline-service,services/notification-service,services/organization-service -am.
  • Full backend test suite cũng đã chạy xanh từ backend: mvn test -q.
  • Test hiện có trong repo: gateway JWT, enrollment service, gradebook aggregation, peer-review aggregate.

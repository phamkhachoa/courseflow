Enterprise Data Platform (dp/) — Kiến trúc hiện tại

Một data platform product-agnostic theo mô hình Open Lakehouse + Medallion + Data Mesh, với control plane governance (data contracts, evidence, Control Tower) và một data plane chạy thật ở local được CI bảo vệ.

Control plane · L4 Data plane (local-live) · L3 Enforcement (smoke) · L2–3 CI/CD gates · L3 Managed cloud deploy · L1

0 Tổng quan & cách đọc bản vẽ

dp tách bạch 2 mặt phẳng: Data plane (dữ liệu thực sự chảy qua) và Control plane (governance bao quanh, quyết định cho phép release hay không). Mọi capability đều phát ra evidence (có ký Ed25519) để Control Tower tổng hợp thành quyết định P0 signoff.

7 Ecommerce Recommendation — dp đã hỗ trợ chưa? Flow tích hợp?

7.3 — Các bước triển khai cụ thể

  1. Onboard product ecommerce: thêm vào products/ + đăng ký source trong platform/ingestion/source-registry.yaml.
  2. Contracts: định nghĩa topic + data-product contracts cho ecommerce.product_viewed / cart_updated / order_completed / recommendation_tracking (clone mẫu recommendation đã có).
  3. Bridge → Bronze: dùng source-bridge-normalize đưa raw event → canonical envelope → Bronze Iceberg (append-only, idempotent).
  4. Silver: conform thành silver.ecommerce_interactions (user_id, item_id, event_type, weight, occurred_at; reduce PII).
  5. Gold: materialize biến thể gold.recsys_interactions cho ecommerce — đây là dataset feature ML đọc.
  6. Governance: cấp access-grant cho ML service (chỉ đọc Gold), bật quality/freshness gate, đăng ký lineage; Control Tower signoff.
  7. ML service (ai/): train trên Gold features (bắt đầu item-CF sẵn có → nâng two-tower/sequential), đẩy qua model registry + activation governance, expose Recommend API.
  8. Feedback loop: impression/click/order phát lại qua recommendation.tracking → Bronze → khép vòng để retrain.
Lưu ý ranh giới: dp là xương sống dữ liệu + governance (ingest, lakehouse, feature handoff, kiểm soát truy cập, lineage). Bản thân thuật toán & online serving là một service riêng ở ai/. Ranh giới sạch này chính là điểm mạnh: đổi model (CF → deep learning two-tower) không đụng tới data platform; thêm product mới (ecommerce) không đụng tới ML.
